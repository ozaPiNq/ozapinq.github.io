Visualizing Django signals using AST analysis

О работе с AST в Python написано немало статей, однако большинство из них ограничивается базовыми примерами, мало применимыми на практике. Официальная документация также не даёт представления о том, что можно делать с помощью модификации и анализа AST. В этом посте я покажу, как можно использовать ast с пользой - с помощью небольшого скрипта построить граф сигналов в Django.

Использование сигналов в Django позволяет создать event-driven, loosely-coupled систему, что очень полезно при работе с большими проектами, состоящими из десятков Django приложений. Однако использование сигналов обладает большим недостатком - практически невозможно понять, что именно происходит в системе при обработке сигнала.

Частично эту проблему можно решить, поняв зависимости между сигналами и их получателями. Для этого необходимо либо использовать статический анализ кода, либо использовать аугментированные сигналы и строить граф в runtime. Как всегда, у каждого из подходов есть свои плюсы и минусы, но в рамках данного поста я буду использовать статический анализ с использованием анализа AST.

Работа с AST

Для работы с AST в Python существует модуль стандартной библиотеки ast, который позволяет распарсить исходный код в AST, а так же предоставляет два полезных класса для анализа и модификации дерева - NodeVisitor и NodeTransformer. Мы для обхода дерева будем использовать NodeVisitor.

При использовании этих классов, нужно определить методы visit_{Node}, где Node - это требуемый нам тип элемента дерева. Полный список можно посмотреть в документации абстрактной грамматики языка.

Для наших целей нужно определить два метода:

  • visit_FunctionDef - для поиска функций-обработчиков синалов, декорированных с помощью @receiver;
  • visit_Assign - для нахождения создаваемых сигналов.

Для построения графа мы будем использовать networkx.DiGraph().

import ast
import networkx as nx
class Visitor(ast.NodeVisitor):
    def __init__(self):
        self.graph = nx.DiGraph()
    def visit_FunctionDef(self, node):
        ...
    def visit_Assign(self, node):
        ...

Можно обойтись только visit_FunctionDef, однако в таком случае не будут видны неиспользуемые сигналы.

Поиск обработчиков сигналов

Каждое определение функции, передаваемое в visit_FunctionDef хранит список декораторов, которыми эта функция обёрнута. Наша задача - выбрать функции, обёрнутые в декоратор @receiver.

Мы проходим по всему списку декораторов в поисках вызова receiver(...).

def visit_FunctionDef(self, node):
    for dec in node.decorator_list:
        if (isinstance(dec, ast.Call) and isinstance(dec.func, ast.Name)
                and dec.func.id == 'receiver'):

Далее мы получаем название аргумента (имя сигнала), который эта функция будет обрабатывать. Сигнал может быть передан качестве аргумента в двух вариантах:

  • ast.Attribute - @receiver(billing_app_signals.my_signal)
  • ast.Name - @receiver(my_signal)

Другие варианты указания сигнала (например, billing_app.signals.my_signal) не учитываются для упрощения примера.

            arg = dec.args[0]
            if isinstance(arg, ast.Attribute):
                signal = arg.attr
            elif isinstance(arg, ast.Name):
                signal = arg.id
            receiver = node.name

После получения названия сигнала и зная название функции-обработчика мы можем добавить их на граф и создать грань между ними.

            self.graph.add_node(signal, role='signal')
            self.graph.add_node(receiver, role='receiver')
            self.graph.add_edge(signal, receiver)

Поиск неиспользованных сигналов

Каждое выражение, передаваемое в visit_Assign хранит значение и цель операции назначения. Определение сигнала выглядит следующим образом:

signal_name = Signal(...)

Нам необходимо найти все операции назначения, в которых значением является вызов (ast.Call) идентификатора Signal и получить название целевой переменной.

Далее полученное значение добавляется в граф.

    def visit_Assign(self, node):
        val = node.value
        if (isinstance(val, ast.Call) and isinstance(val.func, ast.Name)
                and val.func.id == 'Signal'):
            signal = node.targets[0].id
            self.graph.add_node(signal, role='signal')

Заключение

AST может быть полезным для выполнения различных задач по анализу кода, которые сложно было бы решить с использованием других средств (например регулярных выражений).

Этот пример, как мне кажется, хорошо показывает, что AST в Python очень мощный инструмент, умение работать с которым может пригодиться при решении повседневных задач. Достаточно сложная задача решается меньше чем в 30 строк кода - главное понять, что именно мы хотим узнать.

Если вы хотите познакомиться поближе с AST в Python и научиться с ним работать, советую ознакомиться с https://greentreesnakes.readthedocs.io.

Но если вы хотите быстро научиться работать с AST, то советую анализировать небольшие куски кода в консоли с помощью astpretty - отличного модуля для форматирования AST.

Полный пример

import sys
import ast
import networkx as nx
class Visitor(ast.NodeVisitor):
    def __init__(self):
        self.graph = nx.DiGraph()
    def write_graph(self, filename):
        nx.write_gexf(self.graph, filename)
    def visit_FunctionDef(self, node):
        for dec in node.decorator_list:
            if (isinstance(dec, ast.Call) and isinstance(dec.func, ast.Name)
                    and dec.func.id == 'receiver'):
                arg = dec.args[0]
                if isinstance(arg, ast.Attribute):
                    signal = arg.attr
                elif isinstance(arg, ast.Name):
                    signal = arg.id
                receiver = node.name
                self.graph.add_node(signal, role='signal')
                self.graph.add_node(receiver, role='receiver')
                self.graph.add_edge(signal, receiver)
    def visit_Assign(self, node):
        val = node.value
        if (isinstance(val, ast.Call) and isinstance(val.func, ast.Name)
                and val.func.id == 'Signal'):
            signal = node.targets[0].id
            self.graph.add_node(signal, role='signal')
if __name__ == '__main__':
    if len(sys.argv) < 2:
        sys.exit('no source files specified!')
    files = sys.argv[1:]
    visitor = Visitor()
    for fn in files:
        with open(fn) as f:
            module_ast = ast.parse(f.read())
        visitor.visit(module_ast)
    visitor.write_graph('output.gexf')
